# Configuration

The complete annotated template is shipped at [`2.6.1-R2/configuration.template.md`](../2.6.1-R2/configuration.template.md). The ready pair uses identical `topology.toml` copies and separate PowerShell endpoint profiles.

## Endpoint profile

```powershell
@{
    Role             = 'alpha'
    Endpoint         = 'alpha'
    Executable       = 'spacetimedb-standalone-alpha.exe'
    ListenAddress    = '127.0.0.1:3910'
    DataDirectory    = 'data'
    SeedArchive      = '..\seed\alpha-data.zip'
    SeedSha256       = '<approved-64-hex-seed-hash>'
    TopologyFile     = 'topology.toml'
    CpuIndex         = 0
    DatabaseIdentity = '<actual-64-hex-identity>'
}
```

Paths are resolved relative to the starter folder. When `data/` is absent, the shared initializer verifies the configured seed hash before extraction. JWT keys live below the ignored release-local `2.6.1-R2/run/jwt-keys/` directory and are generated by standalone; they are never shipped or committed.

## Topology

```toml
version = 1
machine-id = "one-approved-machine-topology"

[defaults]
protocol-version = 1
queue-capacity-bytes = 65536
max-message-bytes = 4096
delivery = "at-least-once"
acknowledgement = "transaction"
result = "ack-inbox"
overflow = "reject"
handshake-timeout-ms = 15000
request-timeout-ms = 5000
max-in-flight = 64

[windows]
namespace = "local"
principal = "current-user"

[endpoints.alpha]
database-identity = "<actual-alpha-identity>"

[endpoints.beta]
database-identity = "<actual-beta-identity>"

[channels.alpha-beta]
mode = "request-response"
publishers = ["alpha"]
subscribers = ["beta"]
schema-version = 1
schema-hash = "<approved-64-hex-schema-hash>"
durable-source-required = true

[channels.alpha-beta.reducer-allowlist]
beta = ["membus_apply_operation"]
```

## Channel modes

| Mode | Shape |
|---|---|
| `direct` | One publisher, one subscriber, one directed route |
| `request-response` | Request route plus independent reverse response route |
| `fanout` | One independent SPSC route per destination |
| `many-to-one` | One independent SPSC route per source |

The first product flow uses `request-response`.

## Validation rules

- unknown fields and unsupported values fail;
- endpoint and channel names must be explicit;
- database identities must exist in the selected data roots;
- every destination needs a reducer allowlist;
- schema version/hash must match at both peers;
- both processes must resolve the same logical topology;
- relative topology paths passed to standalone are rejected by the topology loader;
- Windows v1 supports `local/current-user` only;
- queue/payload sizes are bounded and validated;
- no default endpoint or fallback is selected.

TOML comments and whitespace do not change the canonical configuration hash, but logical values do.

## Changing the sample

Changing ports or CPU indices only requires endpoint profiles. Changing endpoint names, database identities, channels, schemas, routing, or allowlists requires coordinated topology updates in both folders and matching database/module configuration. Never edit one topology copy alone.
